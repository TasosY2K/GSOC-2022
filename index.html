Plug and Plot with Internet data

Reusable visualizations for Internet open datasets

“Plug and plot”

The building blocks will be developed in the ObservableHQ platform, which uses the popular visualization library D3.js. The building blocks will be open-source code, published as templates in the ObservableHQ platform. Any user would be able to fork/copy a template and plug their own data, and/or modify it according to their needs. Since the code will be in D3.js, the building blocks can be re-usable by any javascript component, not only in ObservableHQ.

Targeted users

Internet researchers and network operators. Internet data: There exist various types of Internet data, mainly, tabular data (e.g., ASN or latency value), timeseries (e.g., AS hegemony per day), graphs (e.g., routing graph), paths (e.g., traceroutes, BGP paths). We will create at least one building block that can visualize each type of data.

Types of plots

Different types of data need different types of visualizations. We will consider a range of visualizations, such as, sankey diagrams (very useful to visualize relations, paths, etc. in Internet data), graphs, geo-maps, distribution plots (e.g., histograms, violin plots), dendrograms, heatmaps.

Implementation & Deliverables

Task 1 - Define data format

Investigate the types of data, the data sources, and the main types of analyses that are of interest to users. Define the data formats to be used. For example, json files with traceroutes (common format for RIPE Atlas data), or json files with IHR data, or lists of BGP paths, etc


Deliverable 1Definition of at least 3 data format types

Task 2 - Exploration of visualization option

Explore all the catalog of data visualization plots offered in Observable and D3.js websites. Investigate what types of plots could fit to each type of the defined data formats. Come up with 2 to 3 suggestions of plot alternatives for every type of data. Also, explore options provided by Observable for filtering, which will interact with the visualizations (e.g., select a time range, or select a part of data)


Deliverable 2Selection of visualization options, and filtering options

For each type of Internet data, as defined above, we will define and implement a group of visuals. Initial ideas for these groups are:

Example: A sankey graph for AS Hegemony

A user selects an ASN, and a sankey graph that depicts the other ASNs that are related to this ASN in the AS-hegemony dataset. In a sankey graph boxes correspond to ASNs and edges to relations between them. Multiple layers of boxes (i.e., more than 2) can be considered to depict ASN relationships of more depth. Coloring can be based on ASN features (e.g., country), and filtering can be used to filter ASNs by specific features.

Example: Graph visualization of ASN relationships in a country

A graph that visualized the AS hegemony of a set of networks with specific characteristics. E.g., a user can input a list of ASNs, or select a list of countries, or select a list of ASN types (e.g., CDNs) and the graph will show all relations between these ASNs from the AS hegemony data, with nodes being the ASNs, edges their connections, and sizes the AS hegemony scores. Hovering over nodes/ASNs will give more information about nodes. Coloring can be based on ASN features (e.g., country), and filtering can be used to filter ASNs by specific features.

Example: Prefix visibility by route collectors

A visualization depicting the visibility of specific prefixes by the different route collectors upon different time points. Users will be able to see how a prefix is seen by different points (different rows or lines) over time (x-axis). Coloring can be used to depict path length, origins ASes, latencies, etc.

Example: Map for RIPE Atlas probes

A geographical map depicting RIPE Atlas probes. There will be filters to select probes of specific types, tags, countries, etc. Also, there will be filters for selecting a list of probes (e.g., used in a measurement). Coloring can be used to depict probe characteristics (e.g., tags) or similarity of probes -if possible to obtain relevant data (e.g., from the AI4NetMon project https://sermpezis.github.io/ai4netmon/ )
